# Day 1 Labs: Machine Learning with AI and Python

Hands-on labs for Day 1. Each one is a small, real-world project, not a snippet drill, and they build on each other, so do them in order. Every lab explains **why** before the **how**, then walks you through step by step.

## Tools

You will use:

- Visual Studio Code
- GitHub Copilot
- Python with the data-science libraries (NumPy, Pandas, scikit-learn, Matplotlib)

Open VS Code, make a folder for the day, and start with Lab 1.

## Labs

| # | Lab | What you practice |
|---|-----|-------------------|
| 1 | [Coding with GitHub Copilot](Lab1-Coding-with-GitHub-Copilot.md) | Build a tested `data_utils.py` module with Copilot: completions, `/explain`, `/fix`, `/tests`, Ask/Edit/Agent modes, and verifying output |
| 2 | [Data Wrangling with NumPy and Pandas](Lab2-NumPy-Pandas.md) | Clean a messy real dataset end to end: types, text, missing values, outliers, grouping, encoding, scaling, and the `X`/`y` split |
| 3 | [A House-Price Estimator with Linear Regression](Lab3-Linear-Regression-House-Prices.md) | An end-to-end regression project: EDA, baseline, cross-validation, residuals, regularization, and saving the model |
| 4 | [Building and Judging a Classifier with Logistic Regression](Lab4-Logistic-Regression.md) | A responsible classification project: precision/recall/F1, ROC and PR curves, thresholds, imbalance, model comparison, and explainability |

## How to work through a lab

1. Read the "Why this lab matters" section first, so the steps have a purpose.
2. Type the code yourself where you can; let GitHub Copilot help, but read every suggestion before you accept it.
3. Stop at each **Checkpoint** and confirm you got the expected result before moving on.
4. Try the **Stretch goals** if you finish early.

## A habit that runs through every lab

Verify what the AI gives you. Running without errors is not the same as being correct: read it, test it on inputs where you know the answer, and keep secrets and private data out of any cloud assistant.
